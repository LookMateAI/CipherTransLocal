功能模块

专注一件事：让本地设备之间传得更直接。

它不是网盘，也不是复杂同步系统。它把日常最常见的跨设备传输动作放在最短路径里，适合临时发截图、收文档、传 APK、同步压缩包。

局域网自动发现

两端打开后自动寻找同一网络里的设备，不需要手动输入 IP，也不用配置复杂连接参数。

文字、图片、文件一起传

临时文字、截图、照片、APK、压缩包都在同一个会话里，传完以后也能回头找。

进度和速度清楚可见

大文件传输时能看到速度、进度和状态，知道是在等待、传输中、暂停还是失败。

暂停、继续、取消

传错文件可以取消，网络不稳可以重试，传输控制不藏在复杂菜单里。

Android 后台更稳

传输时使用前台服务和 Wi-Fi 保活策略，减少退到后台后连接中断的情况。

重要操作二次确认

删除设备、清空历史这类动作会再次确认，避免手滑清掉重要记录。

真实界面

不是概念图，是真正用来传文件的界面。

Android 端把设备、历史和设置做成手机上顺手的结构；桌面端负责稳定收发、后台运行和管理历史记录。

打开后先找设备，传完后还能回头查。 搜索、历史、空状态都围绕真实局域网传输场景设计，不让用户猜当前发生了什么。
设备中心

显示当前发现的 Windows 或 Android 设备，在线状态和刷新动作明确可见。

传输历史

文字和文件记录按时间保留，适合找回刚刚传过的内容。

异常也说清楚

没有发现设备时提示网络、Wi-Fi、防火墙等可能原因，减少无效排查。

使用方式

打开两端，看到设备，直接发送。

使用方式接近日常聊天：先选设备，再发消息或文件。你不需要记 IP，不需要注册账号，也不需要把文件先发到别的应用里。

1

两台设备连同一个网络

电脑和手机连接同一个 Wi-Fi，或处在可互通的局域网里。

2

打开 CipherTransLocal

桌面端和 Android 端会互相广播并发现对方设备。

3

像聊天一样发送

发送文字、图片或文件，进度和结果都会留在对应设备的会话里。
